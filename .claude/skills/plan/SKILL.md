spec 기반으로 구현 계획을 수립하는 스킬이다. 다음 절차대로 진행하라.

## 절차

1. **spec 확인**: `.claude/planning/{브랜치명}-spec.md`를 읽고 요구사항을 재검토하라.

2. **기술적 분석**: 기존 코드와의 연관성, 의존성, 영향도를 파악하라. 테스트 전략도 수립하라.

3. **plan.md 생성**: `.claude/planning/{브랜치명}-plan.md`에 아래 형식으로 저장하라.

4. **승인 대기**: 계획을 사용자에게 보여주고 반드시 승인을 받아라. 승인 없이 구현을 시작하지 마라.

## plan.md 형식

```markdown
# [기능명] Plan

## 개요
- spec 요약
- 기술적 접근 방법

## 구현 단계
1. [ ] 1단계 작업
2. [ ] 2단계 작업
3. [ ] 3단계 작업

## 파일 변경사항
- `경로/파일명`: 변경 내용
- `경로/파일명`: 신규 생성

## 테스트 계획
- Unit Test 목록
- Integration Test 시나리오

## 위험 요소
- 예상 문제점과 대응방안

## 검증 기준
- 완료 확인 방법
```

## 승인 후

승인을 받으면 사용자에게 안내하라:
- "구현해줘"로 plan 체크리스트 순서대로 개발 진행
