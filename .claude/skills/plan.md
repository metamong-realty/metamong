# Plan (구현 계획)

spec을 바탕으로 구체적인 구현 계획을 수립합니다.

## 실행 프로세스

1. **기존 spec.md 확인**
   - `.claude/planning/{브랜치명}-spec.md` 읽기
   - 요구사항 재검토

2. **기술적 분석**
   - 기존 코드와의 연관성 분석
   - 의존성 및 영향도 파악
   - 테스트 전략 수립

3. **plan.md 생성**
   - `.claude/planning/{브랜치명}-plan.md` 에 저장
   - **사용자 승인 대기**

## Plan 문서 형식

```markdown
# [기능명] Plan

## 개요
- spec 요약
- 기술적 접근 방법

## 구현 단계
1. [ ] 1단계 작업
2. [ ] 2단계 작업
3. [ ] 3단계 작업

## 파일 변경사항
- `경로/파일명`: 변경 내용
- `경로/파일명`: 신규 생성

## 테스트 계획
- Unit Test 목록
- Integration Test 시나리오

## 위험 요소
- 예상 문제점과 대응방안

## 검증 기준
- 완료 확인 방법
```

## 승인 후 진행

계획 승인 받은 후:
- "구현해줘"로 실제 개발 진행
- plan.md의 체크리스트를 따라 순차 진행
