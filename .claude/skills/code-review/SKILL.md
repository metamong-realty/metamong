구현된 코드를 5가지 관점에서 검증하라. 변경된 파일들을 모두 읽고 아래 체크리스트에 따라 분석하라.

## 검증 관점

### 1. 코드 품질
- ktlint 규칙 준수 여부
- DDD 원칙 (Entity, VO, Aggregate) 적용 여부
- Clean Architecture 의존성 방향 위반 여부
- SOLID 원칙 준수 여부
- @Query 애노테이션 미사용 확인

### 2. 보안
- 입력값 검증 (@Valid) 적용 여부
- SQL Injection 방지 (QueryDSL/파라미터 바인딩)
- 인증/인가 처리 여부
- 민감정보/에러 정보 노출 방지

### 3. 성능
- N+1 쿼리 문제
- 인덱스 사용 적절성
- 캐시 전략 적용 여부
- 불필요한 객체 생성

### 4. 테스트
- Unit Test 존재 및 통과 여부
- 테스트 커버리지 80% 이상
- Edge Case / 실패 케이스 테스트

### 5. 문서화
- API 문서 (Swagger @Operation, @Schema)
- 복잡한 비즈니스 로직 주석

## 결과 형식

아래 형식으로 리뷰 결과를 작성하라:

```markdown
## 코드 리뷰 결과

### 잘된 점
- 구체적인 칭찬 포인트

### 개선 필요
- 파일명:라인번호 - 개선사항 및 이유, 해결 방법

### 필수 수정
- 보안/성능 관련 중요 이슈

### 제안사항
- 코드 품질 향상을 위한 추가 제안
```

**객관적으로 분석하라. 무조건 칭찬하지 마라. 리뷰나 제안 시 반드시 기술적 근거를 함께 제시하라.**
