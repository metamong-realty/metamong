---

기능 요구사항을 명세하는 스킬이다. 다음 절차대로 진행하라.

## 절차

1. **코드베이스 탐색**: `Task(subagent_type="general-purpose")`로 관련 파일을 탐색하여 현재 구조를 파악하라. 메인 컨텍스트를 오염시키지 마라.

2. **요구사항 명확화**: 모호한 점은 반드시 사용자에게 질문하라. 추측하지 마라.

3. **spec.md 생성**: `.claude/planning/{브랜치명}-spec.md`에 아래 형식으로 저장하라.

## spec.md 형식

```markdown
# [기능명] Spec

## 요구사항
- 핵심 기능 설명
- 상세 동작 정의

## 관련 파일
- 수정 대상 파일 목록

## 결정 사항
- 논의 후 결정된 내용

## 제외 사항
- 이번 작업에서 하지 않는 것
```

## 완료 후

spec 작성이 끝나면 사용자에게 다음 단계를 안내하라:
- `/plan`으로 구현 계획 수립 (권장)
- 또는 현재 세션에서 바로 진행
