# Code Review (코드 리뷰)

구현된 코드의 품질을 검증하고 개선사항을 제안합니다.

## 리뷰 체크리스트

### 1. 코드 품질 
- [ ] ktlint 규칙 준수
- [ ] DDD 원칙 적용 (Entity, VO, Aggregate)
- [ ] Clean Architecture 의존성 규칙
- [ ] SOLID 원칙 준수
- [ ] @Query 애노테이션 미사용 확인

### 2. 보안 검증
- [ ] 입력값 검증 (Validation)
- [ ] SQL Injection 방지
- [ ] 인증/인가 처리
- [ ] 민감정보 노출 방지
- [ ] 에러 정보 노출 방지

### 3. 성능 최적화
- [ ] N+1 쿼리 문제 확인
- [ ] 적절한 인덱스 사용
- [ ] 캐시 전략 적용
- [ ] 불필요한 객체 생성 최소화

### 4. 테스트 검증
- [ ] Unit Test 존재 및 통과
- [ ] Integration Test 시나리오
- [ ] 테스트 커버리지 80% 이상
- [ ] Edge Case 테스트

### 5. 문서화
- [ ] API 문서 (Swagger)
- [ ] 코드 주석 (복잡한 비즈니스 로직)
- [ ] README 업데이트 (필요시)

## 리뷰 결과 형식

```markdown
## 코드 리뷰 결과

### ✅ 잘된 점
- 구체적인 칭찬 포인트

### 🔧 개선 필요
- 파일명:라인번호 - 개선사항
- 이유와 해결 방법 제시

### 🚨 필수 수정
- 보안/성능 관련 중요 이슈

### 📚 제안사항
- 코드 품질 향상을 위한 추가 제안
```
